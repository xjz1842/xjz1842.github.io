2018年规划

现在2018年已经过去两个多月了，才知道去做新的一年的规划，在过去的两三年，我在两家公司带过，一个是互联网行业(做电商的)，一个是传统企业到网联网+医疗的行业，自己从一个啥也不懂的行业小白，成长到了一个会技术框架的开发工程师，但是向精通底层的技术专家迈进的路上，很迷茫，也很努力。且行且珍惜。

2018的技术栈的规划：

  • 源码专题：(3-4月)
    Spring源码，mybatis的源码， SpringMVC的源码的设计模式，框架的设计思想的剖析， spring boot spring cloud的分布式服务的使用，对于分布式服务的限流、降级有更深入的了解，Netty的源码分析。 elastic-job
    sharding-jdbc, Mycat以及Spring cloud的各个组件的分析等。

  • 并发专题：（5月）
    java concurrent packaged下的代码的解析，理解并实现分布式服务下的限流和服务的降级。

  • 微服务专题：（6，7，8月）
    spring cloud and spring boot

  • 工程化专题 （9月）
    K8s docker maven git

  • 分布式rpc专题 （10-11月）
    redis zookeeper dubbo rmi dubbo spring cloud

  • 网络通讯 NIO （12月）
    tcp/ip http mina netty

    1. 阅读五本书 （3-12月）
      <<分布式缓存原理和实现>> <<数据结构与算法>> <<代码整洁之道>> <<重构>> <<分布式服务原理与实践>>

    3 完成自己的最后的学位论文：（3-12月）
    4月份一定要开题，不然真的就没有机会了。争取今年拿到学位。

    1. 坚持写博客，争取开源一个项目，能得到超过一个100个star的项目。（1月）

    2. 自己也会实现写一个属于自己的框架。（1月）
